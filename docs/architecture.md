## 2. 数据流图

### 2.1 日志上链流程

```
堡垒机操作 → JSON日志 → Fluentd采集 → 批次处理(5min窗口)
                                            │
                                            ▼
                                      Merkle树构建
                                            │
                                            ▼
                                      Merkle Root
                                            │
                                    ┌───────┴───────┐
                                    ▼               ▼
                              哈希链追加        Ed25519签名
                                    │               │
                                    └───────┬───────┘
                                            │
                                            ▼
                                  上链存证(record_audit)
                                            │
                                    ┌───────┴───────┐
                                    ▼               ▼
                              链上证明存储      原文存入ES
```

### 2.2 审计验证流程

```
审计员提交Log ID
        │
        ▼
从ES获取日志原文
        │
        ▼
确定所属批次，取回全部日志
        │
        ▼
重建Merkle树，计算本地Root
        │
        ▼
从链上查询该批次存证记录
        │
        ▼
┌───────┴───────┐
▼               ▼
Root比对        Ed25519验签
▼               ▼
┌───────┴───────┐
▼               ▼
 输出综合验证结果
```

## 5. 安全性设计

### 5.1 密钥管理
- Ed25519私钥仅在内存中使用
- 私钥文件权限600
- 生产环境建议迁移至HSM或TEE

### 5.2 签名安全
- 每次签名记录操作日志
- 验证失败时返回明确失败原因

### 5.3 数据完整性
- 所有哈希计算使用SHA-256
- Merkle树构建逻辑可复现
- 批次间哈希链连续性可独立验证

### 5.4 合约安全
- batch_id唯一性校验防止重复上链
- 预留管理员地址权限控制
